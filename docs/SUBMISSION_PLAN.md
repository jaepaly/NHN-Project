## 3. 영상 스토리보드 (30~60초)

> **갱신 2026-07-23** — L3(#101)가 게이트 통과·확정([GATE_DECISION_0728](GATE_DECISION_0728.md))되어 **몽타주 후보에서 영상의 중심축으로 승격**. 편집 원칙: *오늘 들어온 기능을 다 넣지 않는다.* 60초에 5비트가 이미 꽉 찼고, 기능 나열은 논지를 흐린다. **"AI가 메뉴에서 고르는 게 아니라 행동을 설계한다"** 한 문장만 남기는 게 목표.

### 비트 시트

| 구간 | 시간 | 장면 | 왜 이 컷인가 |
|---|---|---|---|
| 콜드 오픈 | 0~7s | 전투 중 `Enter` → 시간이 10%로 늘어짐 → **"얼음 감옥"** 타이핑 → 적이 얼음에 갇힘 | 규칙을 설명하지 않고 보여준다 |
| 논지 | 7~13s | 자막: *"정해진 스킬은 없다. 당신의 문장이 주문이 된다"* | 30초짜리 시청자를 위한 한 줄 |
| **핵심 — 행동 설계** | **13~30s** | **"분신을 만들어서 지그재그로 돌진시켜라"** 를 **한 글자씩 보이게** 타이핑 → 분신 생성 → **지그재그로 2초 → 돌진 1초** 이동. 자막: *"AI가 행동을 설계했다"* | **이 영상의 존재 이유.** 메뉴 선택이 아니라는 유일한 증명 |
| 성장 | 30~42s | 같은 원소 3연속 시전 — 친화가 쌓이며 이펙트가 1→2→3단계로 화려해짐. `COMBO` 자막 | 로그라이크 성장 + 화면이 화려해지는 구간 |
| 기억하는 보스 | 42~54s | 보스: *"지난번엔 화염이었지"* → 화염 저항 → 플레이어가 **다른 원소**로 파훼 | 런을 넘어 기억한다 = 두 번째 차별점 |
| 클로징 | 54~60s | 로고 + *"Built with AI, Powered by AI"* + Pages URL | |

## 6. 심사 기간 Gemini 할당량 정책 (초안 — R2 임재윤, 2026-07-19)

> 최악 케이스: **심사위원이 몰려 무료 티어 일일 한도(RPD)를 소진** → 판정이 429로 실패. 게임은 폴백으로 계속 돌지만 판정 품질이 저하된다.

### 6-1. 현재 사실 (실측·공식 문서 기준)
- 모델: `gemini-flash-lite-latest` 별칭 → 현재 **Gemini 2.5 Flash-Lite**로 해석됨.
  ⚠️ 구 `gemini-2.0-flash-lite`는 **2026-06-01 종료**됨. `-latest` 별칭 덕에 자동 승계되어 무중단. (핀 고정 아님)
- 무료 티어 한도(2.5 Flash-Lite): **15 RPM · 250K TPM · 1,000 RPD**. (지시서의 "RPD 500"은 보수치 — 실제 1,000)
- 유료(PayGo) 가격: 입력 **$0.10 / 1M tokens**, 출력 **$0.40 / 1M tokens** (Batch/Flex 50%↓).

### 6-2. 데모 호출 프로필 (캐시 반영)
| 호출 | 빈도 | 캐시 |
|---|---|---|
| 판정 `/` | 신규 문장당 1회 (동일 문장 반복은 0회) | localStorage, `incant:judge:v2:meaning-v2.1:` |
| 보스 대사 `/boss-line` | 보스전당 1회 | 런 요약 키 |
| 진화·융합 `/evolve-name` | 런당 1~2회 | localStorage, `incant:evolvename:name-v1.0:` |

- **빈 캐시 1회 완주 ≈ 18~28 콜** (대부분 판정, 다수는 반복·캐시로 절감).
- 무료 RPD 1,000 ≈ **일 35~55회 완주** 여력. 다수 동시 접속 시 15 RPM이 먼저 병목.
- 토큰당 비용 극소: 판정 1콜 ≈ 820 tokens ≈ **$0.00014**. **1만 콜 ≈ $1.4**.

> **⚠️ C안(능동형 마나) 채택 영향 (#53-5, 2026-07-21 확인)**: 시전 경제 C안이 **글로벌 쿨다운 3초 하한을 0.4초 인풋락으로 교체**(`startGlobalCooldown` 미사용, `castInputLockSeconds:0.4`). → 마나 넉넉할 때 **버스트 시전율이 Gemini 무료 15 RPM을 초과** 가능 → 그 구간 429 → Mock 폴백(무중단이나 판정 품질 저하). **완충**: 캐시(반복 무료)·마나 게이팅(지속율 제한)·폴백. **결론**: 평상시 문제없음, 마나 버스트 시 일부 Mock. → **§6-4의 심사기간 유료 전환 권고가 더 중요해짐**(유료 RPM↑). 부수: 프록시 `RATE_LIMIT_PER_MIN=20`이 Gemini 15보다 느슨 → 15로 맞추면 신호 명확(사소).

### 6-3. 대응 (노력 순 — 위에서부터)
1. **이미 내장 (코드 0 변경)**: localStorage 캐시 + 2.5초 타임아웃 폴백 + MockJudge. **429여도 게임 무중단**(판정 품질만 저하). ← 최소 안전망 확보됨.
2. **캐시 프리워밍**: 영상·시연용 대표 코퍼스를 사전 판정해 캐시 시드 → 흔한 입력은 호출 0. (심사 직전 `snapshot:quality` 계열로 워밍)
3. **결제 활성화 (권장)**: 무료→유료 전환 시 RPD 한도 대폭 상승, **비용은 사실상 무시 가능**(만 콜 $1.4). 키·코드 불변, 대시보드 토글만.
4. **키 로테이션 (무료 유지 시)**: 무료 키 2~3개를 프록시가 429 시 순환 → 무료로 RPD 2,000~3,000. `worker.js` 소폭 수정 필요.

### 6-4. 권고
- **심사 기간엔 결제 활성화(유료 전환)가 가장 안전·저렴** — 최악 트래픽도 몇 달러. 그 전까진 내장 폴백으로 충분.
- **재현성**: 심사 창 동안 모델을 `gemini-2.5-flash-lite` 명시 핀으로 고정 검토 — `-latest`가 심사 중 자동 갱신되어 판정 거동이 바뀌는 것을 방지 (안정성 vs 자동 승계 트레이드오프).
