## 3. 영상 스토리보드 (30~60초)

| 구간 | 시간 | 장면 |
|---|---|---|
| 콜드 오픈 | 0~8s | 전투 중 Enter → 슬로모션 → "얼음 감옥" 타이핑 → 적이 얼음에 갇힘. 주문명 각인 |
| 문제 제기 | 8~15s | 자막: "정해진 스킬은 없다. 당신의 문장이 주문이 된다" |
| 창의 주문 몽타주 | 15~35s | 서로 다른 3~4개 주문 연속 발동 (원소·폼이 확연히 다른 조합으로 선정) |
| 기억하는 보스 | 35~50s | 보스: "지난번엔 화염이었지" → 화염 실드 → 플레이어가 새 주문으로 파훼 |
| 클로징 | 50~60s | 로고 + "Built with AI, Powered by AI" + Pages URL |

- 녹화: OBS 1080p60 / 편집: 팀 보유 툴 (자막 필수 — 무음 시청 대비)
- 업로드: YouTube 공개(또는 미등록) — 마감 전 링크 접근 테스트

## 4. AI 활용 기술 문서 (제출물 ④) 구성안

**1부 — 게임 속 AI (제품)**
- 주문 판정 아키텍처: 프롬프트 전문, JSON 스키마, 검증·폴백 체인, 캐싱
- 밸런스 가드레일: LLM을 신뢰하지 않는 설계 (클램프·반복 패널티·쿨다운)
- 기억하는 보스: 히스토리 분석 → 내성/카운터, 런 간 기억, 대사 생성
- 인프라: Cloudflare Worker 프록시, 레이트리밋, (옵션) WebLLM 이중화

**2부 — AI로 만든 게임 (프로세스)**
- 3인 × 에이전트(Claude/Codex) 협업 워크플로
- 실제 프롬프트·산출물 사례: 파츠 config 대량 생산, QA 스크립트 생성 등
- AI_USAGE_LOG 통계 (도구별·작업별 활용 비중)
- 배운 점: 에이전트 지휘 노하우 (본선 '에이전트 설계서'와 직결되는 어필 포인트)
