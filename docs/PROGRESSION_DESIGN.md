# 성장 시스템 설계 — 보상 풀·각인·정령·진화 (Phase 3.5)

> 상태: **설계 확정 대기 (총괄 초안 2026-07-16)** · 구현 순서 §6 · 담당 총괄, 리뷰 이도원
> 배경: "보상이 매 런 동일 + 강화 체감 없음 + 뱀서류 무기/펫 성장 재미 부재" (총괄 피드백)

## 0. 설계 원칙 — 영창이 주인공

INCANT의 정체성은 "내가 쓴 문장이 무기". 모든 오토 딜 소스(각인·정령)는 이를 **보조**한다:

1. 오토 소스 슬롯 상한: **각인 2 + 정령 2**
2. 오토 딜 총합 ≤ 수동 영창 지속딜의 **~40%** (밸런스 게이트)
3. 자동 시전은 주문 히스토리에 **기록하지 않음** — 반복 패널티 면제, **보스 기억은 수동 영창만** 반영
4. 성장의 정점(진화·융합)에는 항상 **LLM 작명**이 있다 — 성장 시스템조차 AI-네이티브

## 1. 보상 풀 확장 (①)

매 런 동일한 3택을 폐지. **런 시드 랜덤**(LLM 없음, 런 내 재현 가능)으로 카드 풀에서 3장 추첨.

| 분류 | 카드 | 효과 |
|---|---|---|
| 패시브 | 생명/마나 증폭 | 기존 |
| 패시브 | 원소 친화 | +15% (원소는 랜덤, v2: 이번 런 사용 원소 가중) |
| 패시브 | 신속 영창 | 글로벌 쿨다운 -0.4s (하한 1.0s) |
| 패시브 | 마나 격류 | 마나 재생 +50% |
| 패시브 | 수호 기점 | 방 시작 시 보호막 +30 |
| **각인** | 주문 각인 / 각인 강화 | §2 |
| **정령** | 정령 계약 / 정령 강화 / 융합 | §3 |

- 같은 카드 재획득 = 스택(패시브) 또는 레벨업(각인·정령)
- `RewardKind` 확장은 R1↔R3 계약 변경 — 카드 UI는 제목·설명 렌더링이라 색 매핑만 추가

## 2. 각인(刻印) 주문 (②) — 내 문장이 무기가 된다

- 방 클리어 보상에서 **이번 런에 영창한 주문 중 하나를 각인** → 축소판이 자동 영창 (마나 무소모)
- 강화: Lv1 주기 6s → Lv2 투사체/파도 수 +1 → Lv3 주기 4s·크기 ↑
- 기본 공격(매직 미사일)도 강화 라인: 1→2→3발
- **진화 (④)**: 조건 = 각인 Lv3 + 동일 원소 친화 보유 → LLM이 원문 기반 격상 주문명 작명
  (예: "화염구" → 『멸망을 부르는 태양의 낙하』) + huge 크기·복합 폼 연출. 런당 1~2회 + 캐시

## 3. 정령(펫) (③) — 원소 정령과의 계약

플레이어 궤도를 도는 발광체 (SummonedOrb 비주얼 재사용, 원소 팔레트 글로우).

**공격 정령**: 자기 원소의 하위 마법을 주기 발사
- Lv1: 소형 bolt / 6s → Lv2: **원소 고유 폼** (화염 nova·뇌전 chain·빙결 cage·해류 wave·대지 zone·질풍 bolt속사·광휘 beam·암영 rain) / 4.5s
- Lv3 진화: 자기 원소 강화형 / **융합**(정령 2체 소모): **주+부속성 이중 원소 마법** — 렌더러 `element_secondary` 조합 그대로 활용, 작명은 진화와 동일 LLM 엔드포인트

**유틸 정령** (v1 2종): 치유(주기 HP 펄스) / 수호(주기 보호막, Lv3: 위기 시 2초 무적 — 쿨다운 김)
(v2 후보: 시간=쿨다운 가속, 탐욕=보상 4택)

## 4. 강화 체감 연출 (⑤)

1. **획득 순간**: 카드 → 플레이어 파티클 수렴 + `+20 MAX HP` 부상 텍스트 + HUD 바 성장 애니메이션
2. **누적 성장**: 보상당 발밑 마법진 룬 링 추가, 친화 원소 오라 궤도 파티클
3. **주문 변화**: 친화 원소 주문 파티클 양·크기 증가 + accent 스파크

## 5. R2 연동 (임재윤 요청 사항)

- 프록시에 **`/evolve-name`** 엔드포인트: 입력(원문 or 원소 조합, 컨텍스트) → 격상 주문명 1개 (12자 이내, 검증·폴백 템플릿 필수)
- 각인 진화·정령 융합이 공유. 폴백: `『{원소} 대격변』` 류 템플릿
