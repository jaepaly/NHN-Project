# 런 구조 v2 — 2스테이지·6전투 MVP

> 상태: **Phase 4 확정안** · 목표 플레이 타임 **8~12분** · 장기 목표인 3스테이지 구조는 출시 후 확장
>
> 배경: 현재 3방은 기술 통합용 세로 슬라이스다. 같은 3웨이브 섬멸이 반복되고 보상이 2회뿐이라 각인 Lv3·정령·진화 빌드를 시험하기 어렵다.

## 0. 결정

방을 클리어하고 다음 방으로 이동하는 선형 골격은 유지한다. 이번 마감에는 분기형 맵을 새로 만들지 않는다.
대신 런을 **2스테이지·6전투·5보상**으로 확장하고, 각 방의 적 조합·승리 규칙·위험 요소를 바꾼다.

```text
스테이지 1: 전투방 A → 전투방 B → 스테이지 보스
                                      ↓
스테이지 2: 전투방 C → 엘리트/규칙방 → 최종 기억 보스 → 런 요약
```

- 최종 보스를 제외한 전투 뒤에 보상 3택을 제공한다: **총 5회**
- 일반 전투방은 1~2개의 짧은 웨이브 또는 단일 엘리트 조우로 구성한다
- 한 방 목표 시간 45~75초, 보스 60~90초, 전체 런 8~12분
- 방 전환은 기존 `RunController` 이벤트와 UI를 재사용한다
- 자동 성장의 상한과 보스 기억에는 기존 원칙을 유지한다: 오토 딜 ≤ 수동 지속딜 약 40%, 자동 시전은 히스토리 미기록

## 1. 전투 진행표

| 순서 | 조우 | 핵심 역할 | 구성·규칙 | 보상 |
|---|---|---|---|---|
| 1 | S1 전투방 A | 조작·영창 적응 | 추격자 중심 2웨이브, 마지막에 사수 합류 | 1 |
| 2 | S1 전투방 B | 타깃 우선순위 | 사수+분열체 혼합 2웨이브 | 2 |
| 3 | S1 스테이지 보스 | 1차 빌드 점검 | 명확한 예고가 있는 탄막·소환 2페이즈, 장기 기억 없음 | 3 |
| 4 | S2 전투방 C | 공간 압박 | 실드셋 또는 위험 지대+기존 적 조합 | 4 |
| 5 | S2 엘리트/규칙방 | 완성 빌드 시험 | 엘리트 속성 1개가 붙은 단일 조우 또는 생존 규칙 | 5 |
| 6 | 최종 기억 보스 | 런 결산 | 단기·장기 기억과 빌드 카운터가 패턴에 직접 반영되는 3페이즈 | 없음 |

5회 보상은 `각인 Lv1 → Lv2 → Lv3 → 친화 → 보조 성장` 또는 정령·패시브 혼합 빌드가 형성될 최소 활주로다.
특정 진화를 강제로 보장하지는 않되, 완성 조건에 도달할 현실적인 기회를 준다.

## 2. 방 유형

### P0 — Phase 4에 반드시 포함

1. **섬멸**: 1~2웨이브 전멸. 기존 전투의 기본형
2. **엘리트**: 수가 적은 대신 강화 속성을 가진 적을 격파
3. **보스**: HP 임계치에 따라 패턴과 전장 규칙이 변화

### P1 — 일정이 남을 때

4. **생존**: 30~45초 동안 증원에서 생존
5. **봉인**: 제한 구역 안에서 일정 시간 버티거나 표식을 파괴

P1을 자르더라도 방마다 웨이브 수·조합·엘리트 속성·배경 색조를 달리해 같은 전투의 반복처럼 보이지 않게 한다.

## 3. 적 역할과 엘리트 속성

새 적을 많이 추가하기보다 기존 역할을 조합해 판단할 거리를 만든다.

| 역할 | 현재/신규 | 플레이어가 내려야 할 판단 |
|---|---|---|
| 추격자 | 현재 | 이동 경로와 근거리 광역 사용 |
| 사수 | 현재 | 우선 제거 또는 투사체 회피 |
| 분열체 | 현재 | 광역 타이밍과 추가 개체 처리 |
| 실드셋 | P0 신규 | 실드가 열린 각도·원소에 맞춘 공격 전환 |
| 지원형 | P1 신규 | 주변 적 강화/회복을 끊기 위한 타깃 우선순위 |

엘리트 속성은 한 적에게 **최대 1개**만 붙인다.

- **신속**: 이동·공격 주기 증가, 외곽 잔상으로 표시
- **수호**: 일정 주기 보호막, 파괴 시 긴 빈틈
- **불안정**: 사망 위치에 짧은 예고 후 위험 지대 생성

수치 증폭만 하는 체력 스펀지는 금지한다. 모든 속성에는 눈에 보이는 예고와 대응 창이 있어야 한다.

## 4. 보스 구조

### 스테이지 보스 — 규칙 학습

- 100~60%: 예고된 방사 탄막
- 60~0%: 하수인 소환과 직선 돌진을 교대
- 기억 시스템은 사용하지 않는다. 패턴 학습과 첫 성장 빌드 점검에 집중한다

### 최종 기억 보스 — 플레이 방식에 대한 응답

- **Phase 1 (100~65%)**: 이전 런 최다 원소에 부분 내성, 오프닝 기억 대사
- **Phase 2 (65~30%)**: 이번 런 최다 원소에 강한 내성. 원거리 위주면 돌진·포위, 근거리 위주면 탄막·장판을 강화
- **Phase 3 (30~0%)**: 소환+카운터 패턴을 결합하되 내성은 추가로 늘리지 않는다

페이즈 전환 때 `화염에 대비했다`, `원거리 영창을 꿰뚫어 봤다`처럼 **무엇을 기억해 어떤 패턴이 바뀌었는지 HUD와 대사로 명시**한다.
기억은 수치 저항뿐 아니라 실제 공격 패턴·안전지대·소환 조합을 바꿔야 한다.

## 5. 비주얼·에셋 게이트

전면적인 스프라이트 교체 대신 현재의 기하학적 다크 아케인 스타일을 완성한다.

### Phase 4

- 방·보스방 색조, 위험 지대와 공격 예고선
- 적 역할·엘리트 속성을 구분하는 실루엣·링·글로우
- 각인·정령·진화 획득 파티클과 HUD 성장 애니메이션
- 보스 페이즈 전환 외형·마법진 변화

### Phase 5

- 팔레트·선 굵기·글로우 강도 최종 통일
- UI 아이콘·스크린샷·영상용 연출 정리
- 성능·가독성·색각 구분 QA와 제출 영상·PDF 마감

새 외부/AI 생성 에셋을 채택하면 `ASSET_CREDITS.md`에 원본·프롬프트·선정 근거를 기록한다.

## 6. 구현 경계

1. `RunController`는 스테이지·조우 인덱스와 보상 여부를 관리한다
2. `WaveManager`는 현재 조우 내부의 웨이브만 관리한다
3. 방 구성은 데이터로 선언하고 씬에 `roomIndex === n` 분기를 누적하지 않는다
4. 마지막 기억 보스만 런 종료를 발생시키며, 중간 보스 처치는 보상 후 다음 스테이지로 전환한다
5. 방 전환 시 HP·마나·보호막·각인·정령·친화·히스토리를 유지하고, 새 런에서 전부 초기화한다
6. `SpellSpec`과 판정 프롬프트 계약은 변경하지 않는다

권장 공개 계약:

```ts
type EncounterKind = 'combat' | 'elite' | 'stage-boss' | 'memory-boss';

interface EncounterDefinition {
  id: string;
  stage: 1 | 2;
  kind: EncounterKind;
  rewardAfterClear: boolean;
  waveSetId?: string;
  roomModifier?: 'none' | 'hazard' | 'survival';
}
```
