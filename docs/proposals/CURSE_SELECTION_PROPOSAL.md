# 저주 후보 선택 공통 규칙 제안

> 상태: 구현 전 검토 초안  
> 목적: 복수 저주 후보의 카테고리 선정, 행동 가중치 비교 및 추첨 방식을 공통 계약으로 정의  
> 적용 대상: 침묵·돌풍·주문 반사·금언·암전·폭염 등 모든 저주방

## 1. 선택 구조

한 런의 저주 후보는 다음 순서로 결정한다.

```text
1. 해금되고 활성화된 저주 후보 수집
2. 기존 비율과 정상방 보장 규칙으로 이번 런의 저주방 수 확정
3. 저주방 몫을 활성 카테고리에 최대한 균등하게 배분
4. 균등 분할 후 남는 몫은 런 시드로 카테고리를 섞어 하나씩 추가
5. 실제 몫이 있는 카테고리 안에서 저주별 행동 지표 계산
6. 행동 지표를 공통 의미의 0~1 점수로 정규화
7. 작은 기본 가중치를 더해 가중치 룰렛 실행
8. 카테고리마다 해당 런에 사용할 저주 1종 확정
9. 카테고리 배정 순서를 다시 섞어 실제 저주방에 연결
```

저주방 총량은 카테고리 수가 늘어났다는 이유로 증가시키지 않는다. 한정된 저주방 몫을 활성 카테고리에 가능한 한 균등하게 나누어, 현재 정상방 비율과 런 피로도를 유지하면서 저주 종류의 노출 다양성을 확보한다.

카테고리 몫 배분은 후보 저주의 행동 점수와 무관하다. 특정 행동 축의 점수가 높다는 이유로 같은 카테고리만 계속 등장해 다른 종류의 저주를 경험하지 못하는 상황을 방지하기 위함이다. 행동 가중치는 카테고리가 실제 몫을 받은 뒤 해당 카테고리 내부 후보 선택에만 사용한다.

### 1.1 카테고리 몫 배분 공식

```text
활성 카테고리 수 = K
이번 런의 저주방 수 = N

카테고리별 기본 몫 = floor(N / K)
추가 배분 수 = N % K
```

모든 활성 카테고리에 기본 몫을 먼저 배정한다. 추가 배분 수만큼 활성 카테고리를 런 시드로 섞고, 앞에서부터 서로 다른 카테고리에 한 방씩 추가한다.

활성 카테고리가 3개일 때 결과는 다음과 같다.

| 저주방 수 | 가능한 카테고리 배분 |
|---:|---|
| 1 | `1:0:0`의 순열 중 하나 |
| 2 | `1:1:0`의 순열 중 하나 |
| 3 | `1:1:1` |
| 4 | `2:1:1`의 순열 중 하나 |
| 5 | `2:2:1`의 순열 중 하나 |
| 6 | `2:2:2` |

예를 들어 저주방이 4개이면 `2:1:1`, `1:2:1`, `1:1:2` 중 하나를 런 시드로 결정한다. 특정 카테고리가 항상 추가 몫을 받지 않는다.

현재 6방 런에서 기존 저주방 계산 결과가 2개라면, 세 카테고리 중 두 카테고리가 각각 한 방을 받고 나머지 한 카테고리는 해당 런에 등장하지 않는다. 장기적으로는 각 카테고리가 비슷한 빈도로 제외될 수 있다.

### 1.2 실제 방 배정

카테고리별 몫을 계산한 뒤 카테고리 목록을 다시 결정적으로 섞어 실제 저주방에 연결한다.

```text
몫: movement 2 / casting 1 / element 1
가능한 실제 순서: movement → element → casting → movement
```

가능한 경우 같은 카테고리가 연속 배정되지 않도록 순서를 구성한다. 단, 카테고리 몫의 차이가 커서 연속 배정을 피할 수 없는 경우에는 계산된 몫을 우선한다.

한 카테고리가 여러 방을 받더라도 카테고리 내부 가중치 룰렛은 런당 한 번만 실행한다. 해당 런에서 같은 카테고리에 배정된 방들은 선택된 저주 1종을 공유한다.

```text
movement 몫 2
→ movement 룰렛에서 돌풍 선택
→ 두 movement 저주방 모두 돌풍
```

이는 한 런 동안 행동 카운터의 일관성을 유지하고, 후보별 실제 선택 확률과 플레이테스트 결과를 해석하기 쉽게 만든다.

## 2. 행동 점수 정규화

저주마다 사용하는 원시 행동 지표와 단위가 다르므로 원시값을 직접 비교하지 않는다.

예:

- 침묵: 이동거리
- 돌풍: 유효 위협 회피율
- 암전: 빛·불꽃 영창 사용 비율
- 폭염: 물·얼음·바람 영창 사용 비율
- 주문 반사: 수동 영창 마나 100당 저위력 가중 영창 수
- 금언: 성공 수동 영창 상위 25%의 언령 비용 평균

각 지표는 다음 의미를 공유하는 `0~1` 점수로 정규화한다.

```text
0.0: 해당 습관이 거의 없음
0.5: 일반적인 플레이 수준
1.0: 해당 습관이 매우 강함
```

초기 구현에서는 저주별 임시 `low / typical / high` 기준과 구간 보간을 사용한다. 이 기준은 최종 밸런스가 아니라 플레이테스트 계측을 시작하기 위한 초기값이다.

표본이 부족한 지표는 실제 계산값을 그대로 사용하지 않고 중립값과 혼합하거나 중립값으로 대체한다. 최소 표본의 종류는 지표별로 정의한다.

## 3. 카테고리 내부 가중치 룰렛

정규화 점수가 가장 높은 저주를 무조건 선택하지 않는다. 최고점 고정 방식은 작은 점수 차이를 `선택 100% / 0%`로 증폭하며, 낮은 후보가 실제 플레이에서 영구히 배제될 수 있다.

대신 각 후보의 행동 점수에 매우 작은 기본 가중치를 더한 값을 추첨 비중으로 사용한다.

```text
최종 가중치 = 정규화된 행동 점수 + 기본 가중치

P(후보 i)
= 최종 가중치 i / 카테고리 내 최종 가중치 총합
```

기본 가중치는 선택률을 균등하게 만드는 장치가 아니다. 낮은 점수의 후보에도 최소한의 **이론적 선택 가능성**만 남기는 장치다.

초기 제안값:

```text
기본 가중치 = 0.02
```

두 후보의 행동 점수가 `1.0`과 `0.0`인 경우:

```text
후보 A: 1.02 → 약 98.1%
후보 B: 0.02 → 약 1.9%
```

행동 성향과 가장 관련된 후보가 압도적으로 자주 선택되지만, 다른 후보가 완전히 사라지지는 않는다. 모든 행동 점수가 `0`이면 후보들이 같은 기본 가중치를 가지므로 균등하게 선택한다.

## 4. 후보 자격

기본 가중치는 다음 조건을 모두 만족하는 후보에만 부여한다.

- 현재 진행도에서 해금됨
- 선택된 카테고리에 등록됨
- 구현 및 계약 검증을 통과함
- 기능 플래그 또는 운영 결정으로 비활성화되지 않음

해금되지 않았거나 비활성화된 저주를 기본 가중치만으로 후보 풀에 넣지 않는다.

## 5. 결정적 난수

카테고리 추가 몫 배분, 실제 방 배정 순서와 카테고리 내부 룰렛은 런 시드에 연결된 결정적 난수를 사용한다.

- 같은 런 기억과 시드에서는 같은 결과 재현
- 런 결과가 저장되어 기억이 갱신되면 다음 런에서 새 계획 생성
- 회귀 테스트에서 고정 난수로 선택 결과 검증 가능

난수 소비 순서가 후보 배열 순서의 우연한 변경에 지나치게 민감해지지 않도록 후보 정렬 또는 안정적인 ID 순서를 계약으로 둔다.

## 6. 플레이테스트 계측과 통계 교정

가중치 룰렛은 계산식 정합성에 대한 의존도를 낮추지만 제거하지는 않는다. 특정 지표가 구조적으로 높은 점수를 얻으면 해당 저주의 장기 선택 빈도도 높아진다.

런 단위로 다음 값을 기록한다.

- 저주별 원시 행동 지표
- 지표별 유효 표본 수
- 정규화 전후 점수
- 최종 가중치
- 후보별 이론적 선택 확률
- 선택된 카테고리와 저주
- 이전 런 및 현재 런 식별 정보

수집한 데이터에서 다음을 확인한다.

- 원시 지표의 중앙값
- 주요 백분위
- 정규화 점수 분포
- 카테고리 내부 후보별 선택 빈도
- 특정 후보가 행동 성향과 무관하게 과도하게 선택되는지
- 표본 부족 런에서 선택 결과가 지나치게 요동하는지

목표는 저주별 실제 선택률을 기계적으로 동일하게 만드는 것이 아니다. 서로 다른 행동 습관이 비슷한 수준으로 강할 때 비슷한 행동 점수를 얻도록 정규화하는 것이다.

초기 데이터가 충분히 쌓이기 전에는 기준값을 빈번하게 변경하지 않는다. 특정 저주의 반복이 실제 플레이 습관 때문인지 계산식의 범위 차이 때문인지 구분할 수 있어야 한다.

## 7. 승인 요청 사항

1. 기존 저주방 비율과 정상방 보장 규칙으로 저주방 총량을 먼저 확정
2. 확정된 저주방 몫을 활성 카테고리에 몫과 나머지 공식으로 최대한 균등하게 배분
3. 추가 몫을 받거나 해당 런에서 제외될 카테고리는 런 시드 기반 무작위로 결정
4. 같은 카테고리에 여러 방이 배정되어도 카테고리 내부 룰렛은 런당 한 번만 실행
5. 실제 몫이 있는 카테고리 안에서만 행동 가중치를 사용
6. 최고점 고정 선택을 결정적 가중치 룰렛으로 교체
7. 초기 기본 가중치 `0.02`로 이론적 선택 가능성 보장
8. 저주별 원시 지표를 공통 의미의 `0~1` 점수로 정규화
9. 구현 후 플레이테스트 데이터를 수집해 정규화 기준과 선택 분포를 후속 교정

승인 전에는 기존 저주 선택 코드를 변경하지 않는다.
