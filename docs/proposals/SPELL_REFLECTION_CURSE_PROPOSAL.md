# 주문 반사 저주 설계 제안

> 상태: 구현 전 검토 초안  
> 목적: 총괄 승인 후 별도 브랜치와 후속 PR에서 구현  
> 범위: 주문 반사 저주 단일 항목의 의도, 선정 조건, 피해·치유 규칙 및 검증 기준  
> 공통 선택 규칙: [CURSE_SELECTION_PROPOSAL.md](./CURSE_SELECTION_PROPOSAL.md)

## 1. 제안 목적

`주문 반사`는 낮은 위력의 수동 영창을 여러 번 나누어 사용하는 플레이 습관을 저격하는 영창 카테고리 저주다.

정상적으로 발동한 수동 영창마다 동일한 최대 체력 비례 피해를 부과한다. 플레이어는 남은 체력과 상황을 고려하여 다음 선택을 해야 한다.

- 약한 영창을 반복하지 않고 한 번의 강한 영창으로 전투 해결
- 적이 모이는 시점을 기다려 광역 영창의 효율 극대화
- 여러 행동을 묶은 시퀀스로 영창 한 번의 가치 증가
- 자동 공격·정령·각인을 활용해 수동 영창 횟수 절약
- 체력을 지불하고 지금 즉시 전투를 해결할지 판단

주문 반사는 영창 자체를 금지하지 않는다. 영창 횟수마다 생명력 비용을 부과해 한 번의 영창으로 얻는 가치를 고민하게 만드는 것이 목적이다.

## 2. 저격할 플레이 습관

단순히 수동 영창을 자주 사용하는 플레이어 전체를 대상으로 하지 않는다. 외부 약화가 적용되기 전의 기준 위력이 낮은 수동 영창을, 사용한 마나에 비해 여러 번 나누어 사용하는 성향을 주 지표로 삼는다.

주문 반사는 `casting` 카테고리에 둔다. 같은 카테고리에 후보가 추가되면 공통 가중치 룰렛 규칙에 따라 하나를 선택한다.

## 3. 저위력 반복 지표

### 3.1 영창별 저위력 점수

특정 power 경계 아래를 모두 같은 저위력 영창으로 세면 경계값에서 결과가 급변한다. 따라서 위력이 낮을수록 연속적으로 높은 점수를 부여한다.

```text
저위력 점수
= max(0, (기준 위력 - basePower) / 기준 위력)
```

기준 위력은 현재 power 분포와 플레이테스트 데이터로 확정한다.

판정에는 플레이어가 선택한 영창의 성향을 나타내는 `basePower`를 사용한다.

- 반복 페널티 적용 전
- 격상 원소 약화 적용 전
- 마나 부족 감쇠 적용 전
- 친화도 및 보상 강화 적용 전

외부 시스템 때문에 낮아진 주문을 저위력 영창 습관으로 오인하지 않는다.

### 3.2 전체 마나 사용량 기준 보정

실제 전투 시간 대신, 수동 영창에 실제 소비한 전체 마나를 분모로 사용한다. 같은 마나를 강한 영창 몇 번에 썼는지, 약한 영창 여러 번으로 잘게 나누어 썼는지를 직접 비교하기 위함이다.

```text
저위력 반복 원시 지표
= 저위력 점수 총합 / (수동 영창 실제 소비 마나 / 100)
```

즉 `수동 영창 마나 100당 저위력 가중 영창 수`를 측정한다.

분모에는 다음 마나만 포함한다.

- 정상적으로 발동한 수동 영창이 실제 소비한 마나
- 감쇠 시전이 실제 소비한 마나
- 정상 발동한 cache 및 fallback 영창이 실제 소비한 마나
- 시퀀스 영창이 plan 단위로 실제 소비한 마나

다음 값은 제외한다.

- 침묵 결계 밖의 마나 침식
- 기타 저주 또는 환경에 의한 마나 소멸
- 마나 상한 초과로 사라진 회복량
- 방 전환 및 디버그에 의한 값 변경
- 정령·각인·자동 공격처럼 수동 영창이 아닌 동작

마나 표본이 너무 적은 런은 비율이 과장될 수 있다. 일정량 미만의 수동 영창 마나를 사용한 런은 공통 중립값과 혼합하거나 중립값으로 처리한다. 최소 마나 표본과 정규화 구간은 플레이테스트 계측 후 확정한다.

### 3.3 시퀀스 처리

시퀀스는 behavior별로 나누어 기록하지 않는다.

- plan 전체의 `basePower`로 저위력 점수 1회 계산
- plan 전체의 실제 마나 소비량을 분모에 반영
- 실행된 behavior 수와 관계없이 수동 영창 1회로 기록

복합 영창을 사용할수록 저주 후보 가중치가 부당하게 높아지는 것을 방지하고, 한 번의 영창 가치를 높인다는 저주의 대응법을 보존한다.

## 4. 반사 피해 규칙

### 4.1 피해량

반사 피해는 플레이어 최대 체력에 비례한다.

```text
반사 피해 = 플레이어 최대 체력 × 반사 피해 비율
```

정확한 비율은 구현 후 플레이테스트 조정값으로 둔다. 초기 테스트 범위는 영창당 최대 체력의 `2~5%`로 제안하며, 시작값은 총괄 승인 후 확정한다.

### 4.2 피해 속성

반사 피해는 일반 적 공격이 아닌 `저주 피해`로 취급한다.

- 쉴드 무시
- 일반 피해 감소 및 방어 효과 무시
- 피격 무적 무시
- HP에 직접 적용
- 반사 피해로 사망 가능
- 넉백 없음
- 적 피격으로 인한 히트스톱 없음
- 적 공격 피격 통계에 포함하지 않음

쉴드 영창으로 반사 피해를 돌려막는 플레이를 막기 위해 새로 생성되는 쉴드와 기존 쉴드 모두 반사 피해를 흡수하지 않는다.

### 4.3 판정 순서

```text
1. 영창 판정과 유효성 확인
2. 마나 소비 성공
3. 주문 반사 피해 적용
4. 플레이어 생존 여부 확인
5. 생존한 경우에만 주문 실행
```

반사 피해로 사망하면 주문 효과는 생성되지 않는다. 영창 입력만 했거나 판정에 실패한 경우에는 반사 피해를 적용하지 않는다.

## 5. 공통 치유 감소

주문 반사 방에서는 플레이어가 받는 모든 체력 회복 효과를 `50%` 감소시킨다.

```text
실제 체력 회복량 = 원래 체력 회복량 × 0.5
```

적 공격으로 잃은 체력과 반사 피해로 잃은 체력을 구분하지 않고, 저주방이 활성화된 동안 발생하는 모든 치유에 같은 배율을 적용한다.

- 회복 영창
- 보상·아이템·오브젝트에 의한 회복
- 패시브 및 기타 시스템 회복
- 이후 추가되는 모든 일반 체력 회복 경로

저주방이 종료되면 치유 감소를 즉시 해제한다. 방 클리어 후 보상 단계에서 이루어지는 회복은 저주가 이미 종료됐다면 정상 수치로 적용한다.

## 6. 영창별 적용 여부

| 상황 | 반사 피해 |
|---|---:|
| 정상 발동한 수동 영창 | 1회 |
| 감쇠 시전 | 1회 |
| cache 결과가 정상 발동 | 1회 |
| fallback 결과가 정상 발동 | 1회 |
| 시퀀스 영창 | plan 전체당 1회 |
| 마나 부족 등으로 실제 발동 실패 | 미적용 |
| 침묵 등 다른 규칙이 먼저 차단 | 미적용 |
| 판정 실패·불발 | 미적용 |
| 정령 공격 | 미적용 |
| 각인 공격 | 미적용 |
| 기본 공격·자동 공격 | 미적용 |

영창의 form, element 및 behavior 수는 반사 피해 횟수를 늘리지 않는다.

## 7. 시각 및 UI

### 7.1 방 진입 안내

```text
저주: 주문 반사
영창할 때마다 생명력이 감소하며, 치유 효과가 절반으로 감소합니다
```

쉴드가 반사 피해를 막지 못하고 반사 피해로 사망할 수 있다는 사실은 상세 규칙 또는 첫 반사 피해 안내에서 분명하게 전달한다.

### 7.2 영창 전 위험 표시

- 영창 UI에 예상 반사 피해량 표시
- 현재 HP가 예상 반사 피해 이하라면 치명 위험 색상으로 표시
- 반사 피해가 쉴드를 무시한다는 아이콘 또는 짧은 문구 제공
- 반사 피해 발생 시 일반 적 피격과 구분되는 저주 색상 연출

영창을 실행하면 사망할 수 있다는 정보를 입력 확정 전에 확인할 수 있어야 한다. 별도 확인 창은 전투 흐름을 끊으므로 사용하지 않는다.

## 8. 기존 시스템과의 상호작용

- 침묵 차단 등 영창 이전 규칙이 반사보다 우선한다.
- 감쇠 시전은 실제 발동하므로 반사 피해를 받는다.
- 마나 환류는 주문이 실행되고 적 처치가 발생했을 때만 정상 적용된다.
- 반사 피해로 사망해 주문이 실행되지 않으면 환류도 발생하지 않는다.
- 시퀀스는 반사 피해 후 생존했을 때만 실행하며, 실행 도중 behavior마다 추가 피해를 주지 않는다.
- 반사 피해는 적 공격 회피율과 피격 통계에 영향을 주지 않는다.
- 치유 감소는 저주방 활성 상태를 단일 기준으로 모든 회복 경로에서 소비한다.

## 9. 플레이테스트 항목

1. 저위력 영창 반복보다 강한 영창 및 복합 시퀀스가 유의미하게 유리한가
2. 반사 피해량이 영창을 금지하지 않으면서 횟수 선택을 요구하는가
3. 쉴드 무시가 UI에서 충분히 전달되는가
4. 반사 피해 사망과 주문 미실행이 억울한 버그가 아니라 규칙으로 이해되는가
5. 치유 50% 감소가 회복을 무효화하지 않으면서 돌려막기를 억제하는가
6. 감쇠·cache·fallback·시퀀스가 각각 영창당 정확히 한 번만 반사되는가
7. 침묵 마나 침식 등 비영창 마나 변화가 선정 지표에 섞이지 않는가
8. 낮은 마나 사용량의 런에서 지표가 과도하게 높아지지 않는가
9. 실제 소비 마나 기준이 의도대로 저위력 분할 사용 습관을 구분하는가
10. 반사 피해와 치유 감소가 결합되어 저체력 입장 시 지나치게 가혹하지 않은가

## 10. 승인 요청 사항

1. 주문 반사를 저위력 수동 영창 반복을 저격하는 `casting` 카테고리 저주로 추가
2. 반사 피해를 최대 체력 비례로 계산
3. 반사 피해가 쉴드·방어·피격 무적을 무시하고 사망 가능
4. 반사 판정 후 사망하면 주문을 실행하지 않음
5. 저주방 활성 중 모든 체력 회복 효과 50% 감소
6. 정상 발동한 수동 영창과 시퀀스 plan당 반사 피해 1회
7. `수동 영창 실제 소비 마나 100당 저위력 가중 영창 수`를 후보 원시 지표로 사용
8. 침묵 마나 침식 등 비영창 마나 변화는 지표에서 제외
9. 반사 피해 비율·최소 마나 표본·저위력 기준 power는 플레이테스트 계측 후 확정

승인 전에는 구현에 착수하지 않는다. 승인 과정에서 변경된 항목을 이 문서에 반영한 뒤 별도 브랜치에서 구현한다.
