- 성장 시스템 ① 보상 풀(시드 랜덤·신규 패시브 3종)과 R2 `/evolve-name`(배포·검증 완료)은 선행 병합됨

설계 원칙 §0(영창이 주인공)은 절대 컷 불가: 오토 슬롯 각인 2+정령 2, 오토 딜 ≤ 수동 ~40%,
자동 시전은 주문 히스토리 미기록(보스 기억은 수동 영창만).

## 2. 트랙 1 — 이도원 (R1) 🎨

### 2-1. chain·cage 폼 렌더러 (P0)

정령 Lv2 고유 폼(뇌전 chain·빙결 cage)과 진화 연출이 의존하는 **선행 작업**이다.
현재 12폼 중 6폼 구현(bolt/beam/wave/nova/zone/rain) — 기존 `areaSpellConfig`/`spellRenderer` 패턴 그대로:

- **chain**: 첫 타깃 적중 후 가까운 적으로 2~3회 연쇄 (뇌전 은유 — 연쇄마다 피해 감쇠 권장, 수치는 임시값 기록)
- **cage**: 대상 주변 감금 링 — 지속 시간 동안 이동 봉쇄 또는 강둔화 (기존 control 둔화 파이프라인 재사용 가능)
- 팀 컨벤션 회귀 스크립트 + Mock 브라우저 검증. 판정 스키마의 `form` enum엔 이미 있으므로 계약 변경 없음

### 2-2. 런 구조 v2 (P0, chain·cage 이후)

[RUN_STRUCTURE_V2.md](RUN_STRUCTURE_V2.md)의 데이터 기반 2스테이지·6전투 경로를 구현한다.
현재 3방 세로 슬라이스를 단순 복제하지 말고, 5회 보상·중간 보스·엘리트 규칙방·최종 기억 보스를 분리한다.
방 전환과 보상 이벤트는 기존 `RunController`, 방 내부 웨이브는 `WaveManager` 경계를 유지한다.

### 2-3. 여유 시 (P1, 순서대로)

1. 보스전 전용 BGM (Phase 3 컷 이월 — 기존 AI 생성 워크플로·에셋 경로 정책 그대로)
2. 스테이지·보스방 배경 색조 변화 (Phase 3 컷 이월)
3. wall/orbit 폼

성장 시스템의 보상·전투 접점 PR 리뷰도 R1 담당 (PROGRESSION_DESIGN 리뷰어).

성장 시스템 ②각인 → ③정령 → ④진화·융합(`getEvolvedName` 소비) → ⑤연출을 순서대로 구현한다

## 5. 병합 순서·경계

1. R1 chain·cage는 **독립 선행 가능** — ③ 정령 Lv2 병합 전 완료 목표
2. `RewardKind` 확장(각인/정령 카드 추가)은 R1↔R3 계약 변경 — 총괄이 계약 커밋을 먼저 올리고 R1 리뷰 후 구현 진행 (Phase 2 계약 우선 패턴)
3. `SpellSpec` 변경 없음 예정 — 필요 시 구현 전 합의
4. 라이브 Gemini 최소: 개발은 `VITE_JUDGE_MOCK=1` + 템플릿 폴백, evolve-name 라이브는 ④ 결합 검증에서 1~2회만
