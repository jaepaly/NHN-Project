## 2. 트랙 1 — 이도원: 에셋·디자인 🎨🔊

> 코드가 아니라 **생성 프롬프트가 원천 소스**인 트랙이다. 모든 생성 시도(도구·프롬프트·채택 여부)를
> AI_USAGE_LOG에 남기면 그대로 제출물 ④의 "AI로 에셋 만들기" 사례가 된다.

### 2-1. 사운드 (P0) — AI 생성 우선

**도구 추천 (무료 티어부터)**
- SFX: **ElevenLabs Sound Effects**(텍스트→효과음, 무료 크레딧), Stable Audio
- BGM: **Suno** 또는 Stable Audio (루프 가능 트랙)

**제작 목록** — `assets/audio/`에 저장, 파일명 규칙 `sfx-cast-fire.ogg` / `bgm-combat.ogg`:

| 분류 | 항목 | 프롬프트 시작점 (영어로, 길이 명시) |
|---|---|---|
| 원소 발동음 8종 | fire/water/lightning/ice/earth/wind/light/dark | "short magical **fire** spell cast, whoosh with crackling embers, arcade game SFX, 0.6s" — 원소마다 질감 단어 교체(물=splash/flow, 번개=zap/crackle, 얼음=crystal/shatter…) |
| 공통 SFX | 타격, 적 처치, 불발(피식), 영창 모드 진입, 보상 선택, 방 클리어, 보스 등장 스팅어 | "soft magical fizzle, failed spell, comedic, 0.4s" 등 |
| BGM | 전투 루프 1곡 (필수), 보스전 1곡 (여유 시) | "dark synthwave loop, mysterious arcane atmosphere, 100 BPM, seamless loop, instrumental, 90 seconds" |

**에이전트 활용 순서 (권장 워크플로)**
1. Codex/Claude에게 위 표를 주고 **"원소별 질감 단어를 반영한 프롬프트 20개 변형 생성"** 요청 → 생성 도구에 넣고 각 2~3개 후보 뽑기
2. 후보 중 채택본만 `assets/audio/`에 넣고, **채택/탈락 사유 1줄**을 AI_USAGE_LOG에 기록
3. 통합 코드는 에이전트에게: "Phaser 3 사운드 매니저 모듈을 만들어라 — preload에서 로드, `playCast(element)`/`playSfx(name)`/`playBgm()` API, 전역 볼륨 0.5, **M키 음소거 토글**, localStorage에 음소거 상태 저장" → 씬 연결 지점은 `castSpell` 호출부·적 처치·UI 이벤트
4. 품질 기준: 발동음은 0.3~0.8초(길면 전투 리듬 깨짐), 볼륨 정규화(피크 -6dB 부근), ogg/mp3 128kbps
5. **AI 결과가 계속 별로면 그 항목만** freesound.org(CC0 필터)·Kenney Audio로 대체

**라이선스 (중요)**: 생성 도구 약관에서 상업/공모전 사용 가능 여부 확인하고,
`docs/ASSET_CREDITS.md`를 만들어 항목별 출처(도구명/프롬프트 또는 샘플 URL/라이선스)를 기록할 것 — 심사 리스크 방지.

### 2-2. 타이틀 화면 (P0)

- `TitleScene` 신설: 어두운 배경 + **INCANT 로고** + "정해진 스킬은 없다. 당신의 문장이 주문이 된다" 카피 + "Press Enter"
- 로고는 **코드 타이포 우선** (웹폰트 + CSS/Phaser 글로우 — 우리 아트 디렉션과 일치, 수정 쉬움). 이미지 생성 AI는 배경 장식(마법진·성운 등)에만
- 시작 → 기존 런 흐름 진입. 데모 영상의 **첫 컷**이 이 화면이 된다
- 에이전트 지시 예: "Phaser 3 TitleScene을 만들어라 — 씬 등록·전환, 로고 텍스트에 글로우 펄스 트윈, Enter/클릭 시 proto 씬 시작, 960×640 FIT"

### 2-3. 웹폰트·파비콘·OG (P1)

- 주문명 각인용 세리프 폰트 (Google Fonts에서 OFL 라이선스 확인 — 예: Noto Serif KR) + 로고용 폰트 → `announceSpell`·타이틀에 적용
- 파비콘(코드 글리프 또는 단순 아이콘) + OG 메타(og:title/description/image) — 링크 공유 첫인상
- 스테이지·보스방 배경 색조 변화는 **여유 시** (Phase 4로 밀어도 됨)

**PR 분리 권장**: ① 사운드+통합 ② 타이틀+폰트+메타. 리뷰는 총괄.

## 3. 트랙 2 — 임재윤: 보스 기억·대사 (R2)

- [ ] **내성 프로필 모듈**: `bossMemory()` 요약 → 최다 원소 저항(피해 ×0.3)·최다 폼 카운터 유형 결정. 순수 함수 + 회귀 스크립트 (전투 적용은 총괄의 보스 코어가 소비)

- [ ] **보스 대사 생성**: 프록시에 `/boss-line` 엔드포인트 — 입력(런 요약)→ 1~2문장 위협 대사. **폴백 템플릿 대사 필수** (프록시 실패 시에도 보스는 말한다), 대사도 검증·길이 제한
- [ ] 인터페이스는 작은 계약 파일로 공개 (`src/spell/` 또는 `src/run/` 소유 폴더) — 총괄 보스 코어·R3 UI가 소비
