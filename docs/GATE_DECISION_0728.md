## 게이트별 상세

### #101 소환 행동 DSL → **승격 확정** ✅

**게이트 조건 3개 전부 충족:**
- 실 Gemini 수율 ≥70% → **100% (20/20)**
- 기존 회귀 안 깨짐 → **전부 통과** (위 근거 1)
- auto ≤40% 게이트 → **유지** (engrave/spirit DPS 게이트 녹색)

R2 의존 항목(프롬프트 `behavior` 출력 + 실LLM 검증)은 **둘 다 완료·머지**(#106·#107). 더 만들 게 없고 이미 main에서 초록 — **7/28엔 승격 도장만.**

이 게임의 **킬러 데모**다("AI가 메뉴에서 고르는 게 아니라 **행동을 설계**한다"). 실질 후속은 **영상 스토리보드에 소환+행동 장면을 반드시 넣는 것**(제출물 ②). 안전벽(화이트리스트 kind + 수치 클램프 + 실패 시 기본 행동 폴백)이 있어 데모 중 이상 출력이 나와도 게임은 항상 돌아간다.

### #67 능동형 마나 → **이미 승격, 프리즈 전 2건만 마무리**

컷/승격 게이트가 **아니다** — C안은 #53에서 채택 확정, #66 머지로 **이미 시전 경제의 베이스**. 7/28의 실제 결정은 "정식화 체크리스트 중 콘텐츠 프리즈(~8/2) 전에 꼭 닫을 것":

- **필수(빠름)**
  - [ ] **보상 풀 충돌 해소** — `신속 영창`(글로벌 쿨다운 감소)이 C엔 **글로벌 쿨다운이 없어 무의미**. 대체(입력락 -0.1s 등)나 폐기 확정. → **총괄 이번 주 처리**
  - [ ] **auto ≤40% 실런 재측정** — 회귀는 녹색이나 수급 방식(크리스탈 능동 회수)이 바뀌어 실플레이 1회 실측 권장. → 총괄
- **연기 가능(데모 후)**: HUD 정식 연출(드롭 텔레그래프·흡수 연출·마나 부상 텍스트), GDD/PROGRESSION_DESIGN 반영.

### #83 판정 폴백 → **B(현행 Mock 폴백) 유지**

이슈의 3택 중 **B**. 근거가 뒤집혔다: "폴백 7%"는 **구모델(3.1)** 데이터였고, #107의 3.5-flash-lite 교체 뒤 실측 폴백은 **0/10**(위 근거 3). 폴백이 드무니:

- **A(타임아웃 2.5→3초)**: 불필요. 폴백이 이미 드물어 이득 없이 느린 판정 때 대기만 늘린다.
- **C(유료 LLM)**: 불필요 + **회피 대상**. 임재윤이 **결제 등록해야 하는 소유자·재무 액션**이라, 데이터가 필요를 말하지 않는 한 하지 않는다.
- **B(현행)**: **비용 0, 작업 0.** 드문 폴백은 원소가 맞는 Mock으로 무중단 처리되고, 다음 시전은 다시 gemini.

데모 당일 안전판: 예시 주문 몇 개를 미리 태워 **localStorage 캐시 프리워밍** 정도만 팁(선택).

### #97 조합 엔진 (에픽) → **#101까지로 범위 동결**

①상태이상·②소환 다양성 머지 완료(#104), ⑤프론티어=소환 DSL(#101) 승격. **나머지는 동결:**

- **③ 이색 효과**(흡혈·반사·중력·즉사·도발), **④ 지형 다양성**(가시/얼음/독 벽) → **제출 후 스트레치**.
- 이유: 각 슬라이스가 **판정·엔진·렌더 3층을 동시에** 건드려 비용이 크고, 관련 이슈들이 "제출물(8/5~) 시간 침범 금지"를 반복 명시(PHASE_5 §4).
- **슬랙이 남을 때만**: ③에서 **흡혈 1종**(피해 X% 회복 — 최고 임팩트·최저 비용)만 열어볼 여지.
