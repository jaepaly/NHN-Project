## 1. 개요

### 1.1 가칭 (팀 투표로 확정)

| 후보 | 의미 |
|---|---|
| **INCANT (인캔트)** | "영창하다". 짧고 강하며 네온 로고와 어울림 |
| **아케인 신택스 (Arcane Syntax)** | "마법의 문법". CS 전공 팀 정체성 반영 |
| **영창 (詠唱)** | 한국어 정체성. 국내 심사에 어필 |

### 1.2 한 줄 컨셉

> **정해진 스킬은 없다. 당신이 입력한 문장이 곧 주문이 된다.**
> 자유 텍스트 주문을 LLM이 판정해 실시간 이펙트로 구현하는 로그라이크 아레나. 보스는 당신이 쓴 주문을 기억하고 대비한다.

### 1.3 장르 / 플랫폼

- 장르: 로그라이크 아레나 서바이벌 + AI-네이티브 주문 창작
- 플랫폼: 웹 브라우저 (데스크톱 우선, GitHub Pages 배포)
- 플레이 타임: 1런 10~15분

### 1.4 셀링 포인트 (심사 어필 지점)

1. **AI-네이티브**: LLM을 빼면 게임이 성립하지 않는다. 주문 판정 자체가 코어 메커니즘
2. **무한한 콘텐츠, 유한한 QA**: 판정(LLM)은 무한하지만 렌더링은 유한 파츠 조합 → 완성도 보장
3. **기억하는 보스**: 업계 난제인 '기억(persistence)'을 게임플레이로 구현
4. **다시 보고 싶은 영상**: 매 플레이가 다른 주문 → 30~60초 영상에 최적

## 2. 코어 루프

```
런 시작 → [방 진입 → 웨이브 전투 → 보상 선택] × 2~3 → 스테이지 보스 → 다음 스테이지
                                                              ↓ (3스테이지 후)
                                     최종 보스 (기억하는 보스) → 클리어 / 사망 → 런 요약 → 재도전
```

- **방(Room)**: 탑다운 아레나. 적 웨이브 2~3회 격퇴 시 클리어
- **보상 선택**: 3택 1 — 마나 최대치 증가 / 원소 친화(특정 원소 위력↑) / 유물(규칙 변경 패시브)
- **스테이지**: 3개. 스테이지마다 적 구성·배경 색조 변화, power 상한 상승
- **사망**: 런 종료. 사용한 주문 히스토리 요약 화면("이번 런의 주문서") → 재도전 유도

## 3. 주문 시스템 (핵심 기믹)

### 3.1 플레이 흐름

1. 플레이어는 WASD로 이동, 기본 공격은 자동 (약한 매직 미사일)
2. **Enter** → 시간이 10%로 느려지는 **영창 모드** + 입력 바 표시
3. 자유 텍스트 입력 (예: "번개를 품은 해일") → Enter
4. LLM 판정 (목표 1.5초 내, 영창 이펙트로 대기시간 연출)
5. 판정 JSON → 파츠 조합 렌더링 → 발동 + 주문명 화면 각인

### 3.2 판정 스키마 (LLM 출력)

```jsonc
{
  "name": "뇌전해일",              // LLM이 지은 주문명 (화면 에코)
  "element_primary": "water",     // 8종: fire, water, lightning, ice, earth, wind, light, dark
  "element_secondary": "lightning", // 옵션 (null 가능)
  "form": "wave",                 // 12종: bolt, beam, wave, nova, rain, wall, cage, orbit,
                                  //        summon, buff, zone, chain
  "size": "large",                // small / medium / large / huge
  "speed": "slow",                // slow / normal / fast
  "status": ["shock", "knockback"], // 상태이상 풀: burn, freeze, shock, slow, knockback, weaken
  "power": 72,                    // 0~100. 창의성·구체성·서사성 판정
  "cost": 45,                     // 마나 비용. power에 비례하되 LLM이 조정
  "flavor": "심연이 하늘로 치솟는다" // 짧은 플레이버 텍스트 (옵션)
}
```

### 3.3 판정 원칙 (프롬프트 설계 방향)

- **구체성 보상**: "불" < "화염구" < "태양의 파편을 뜯어낸 겁화" — 묘사가 풍부할수록 power↑
- **의미 정합성**: 문장의 의미와 element/form이 일치해야 함 (판정의 재미 = 게임의 재미)
- **장난 입력 처리**: 의미 없는 입력("ㅁㄴㅇㄹ")은 power 5의 '불발' 주문으로 판정 — 실패도 연출로 소화
- **금칙 처리**: 부적절 입력은 안전한 기본 주문으로 변환 (심사 데모 안전성)

### 3.4 밸런스 가드레일 (엔진이 강제, LLM을 신뢰하지 않음)

| 가드레일 | 규칙 |
|---|---|
| 마나 경제 | 마나 자동 회복(초당 n). cost > 보유 마나면 영창 실패 |
| power 상한 | 스테이지별 캡 (S1: 60, S2: 80, S3: 100). LLM이 초과값을 줘도 클램프 |
| 반복 패널티 | 동일/유사 주문 반복 시 power 20%씩 감소 (다양성 유도 → §4 보스 기억과 시너지) |
| 쿨다운 | 영창 후 3초 글로벌 쿨다운 (스팸 방지 + API 비용 통제) |

### 3.5 안정성 (버그 제로 전략)

- **스키마 검증**: enum 밖 값 → 1회 재시도 → 실패 시 기본 주문 폴백. 렌더러에는 검증된 값만 도달
- **타임아웃**: 2.5초 초과 시 로컬 키워드 판정기(MockJudge)로 즉시 폴백 — 게임이 멈추는 경우는 없음
- **캐시**: 동일 문장 = 동일 판정 (localStorage). 데모 시연 시 응답 속도 보장
- **판정기 추상화**: `SpellJudge` 인터페이스 — `MockJudge`(키워드 결정론) / `GeminiJudge`(프록시) / (W4) `WebLLMJudge`

## 4. 기억하는 보스

### 4.1 런 내 기억 (Short-term)

- 보스는 해당 런의 주문 히스토리(JSON 배열)를 참조
- **원소 내성**: 최다 사용 원소에 실드 생성 → 그 원소 피해 70% 감소. 화면에 명시적 표시("보스가 화염에 대비했다")
- **카운터 패턴**: 플레이어가 원거리 폼(bolt/beam) 위주면 돌진형, 근거리 폼(nova/zone) 위주면 원거리형 패턴 선택
- **디자인 의도**: 반복 패널티(§3.4)와 함께 "매 순간 새로운 주문을 창작해야 이긴다"는 핵심 경험 완성

### 4.2 런 간 기억 (Long-term, localStorage)

- 보스가 이전 런들의 데이터(사망 횟수, 애용 원소, 최고 피해 주문)를 기억
- 재도전 시 오프닝 대사 변화: *"또 왔군. 지난번엔 '뇌전해일'이었지. 이번엔 통하지 않는다."*
- 첫 페이즈부터 이전 런 최다 원소에 부분 내성 → 플레이어는 매 런 다른 빌드 강요
- LLM으로 보스 대사를 히스토리 기반 생성 (판정과 동일한 인프라 재사용)

## 5. 전투 / 적 디자인

- **플레이어**: HP 100, 마나 100(자동 회복), 이동 + 자동 기본 공격 + 영창
- **적 (기하 도형 + 네온 글로우)**:
  | 적 | 형태 | 행동 |
  |---|---|---|
  | 추격자 | 삼각형 | 플레이어 직선 추격 |
  | 사수 | 사각형 | 원거리 탄막 |
  | 분열체 | 육각형 | 처치 시 소형 2기로 분열 |
  | 실드셋 | 원+링 | 특정 원소만 통하는 회전 실드 (원소 다양성 강제) |
- **스테이지 보스**: 대형 기하 조형물. 최종 보스만 기억 시스템 풀가동, 스테이지 보스는 간소화 버전

## 6. 렌더링 아키텍처 — 유한 파츠 조합 엔진

```
판정 JSON → [① form: 궤적·판정 로직] × [② element: 파티클 스킨·팔레트]
          × [③ secondary: 오버레이] × [④ size/speed: 스케일링]
          × [⑤ status: 상태이상 연출] + [⑥ name: 주문명 타이포 연출]
```

- 8원소 × 12폼 = 96 기본 조합 + 보조원소/크기/상태이상 변주 → 수만 가지 시각 결과
- 각 조합은 Phaser 파티클 emitter config(데이터)로 정의 → **에이전트로 대량 생산, 사람은 큐레이션**
- QA: 96 조합 자동 스크린샷 스크립트로 전수 검사 (무한 게임, 유한 QA)

## 7. 아트 디렉션 — 네온 아케인 미니멀

- 배경: 짙은 남색~검정 그라데이션 + 희미한 마법진 그리드
- 개체: 순수 기하 도형 + 강한 글로우(bloom). 스프라이트 에셋 최소화
- 원소 팔레트: fire=주황적, water=청록, lightning=전기청자, ice=백청, earth=호박갈, wind=연녹, light=금백, dark=보라흑
- 폰트: 주문명 각인용 세리프(마법서 느낌) + UI용 산세리프
- 사운드: 원소별 발동음 + 영창 모드 저역 드론. 무료 라이선스 + AI 생성 병행

## 8. LLM 아키텍처

```
[브라우저 (GitHub Pages)]
   ├─ SpellJudge 인터페이스
   │    ├─ GeminiJudge ──HTTPS──▶ [Cloudflare Worker 프록시] ──▶ Gemini Flash API
   │    │                          (키 은닉, 레이트리밋, CORS)
   │    ├─ MockJudge (키워드 결정론 — 로컬 개발·장애 폴백)
   │    └─ WebLLMJudge (W4 옵션 — Qwen3 브라우저 구동, 오프라인 모드)
   └─ 캐시 (localStorage: 문장 해시 → 판정 JSON)
```

- Gemini Flash 무료 티어로 개발/시연 커버, 프록시에서 IP당 레이트리밋
- 보스 대사 생성도 동일 프록시 경유 (엔드포인트 분리: `/judge`, `/boss-line`)

## 9. 기술 스택

| 영역 | 선택 |
|---|---|
| 게임 | TypeScript + Vite + **Phaser 3** |
| 배포 | GitHub Actions → GitHub Pages (push 시 자동) |
| 프록시 | Cloudflare Workers (`proxy/`, wrangler) |
| LLM | Gemini Flash (판정·보스 대사) / WebLLM Qwen3 (옵션) |
| 협업 | GitHub PR 기반, main 보호, 전원 에이전트 사용 + AI_USAGE_LOG 기록 |

## 10. 스코프 가드 (잘라도 게임이 성립하는 것들)

우선순위가 밀리면 아래 순서로 컷한다. **코어(주문 창작 전투 + 기억 보스)는 절대 컷 불가.**

1. WebLLM 오프라인 모드 (W4 옵션)
2. 유물(패시브) 종류 축소 (9종 → 3종)
3. 스테이지 3 → 2 축소 (보스 직행)
4. 적 4종 → 3종
5. 런 간 기억 대사 생성 → 템플릿 대사로 대체
