## 기록 규칙

- 모든 작업 항목과 페이즈 제목에 `[R1]` 태그를 붙인다.
- 새로운 R1 작업은 기존 내용을 덮어쓰지 않고 페이즈 또는 작업 묶음 단위로 아래에 추가한다.
- R1에 계획된 역할과 R1이 실제 수행한 작업을 구분해 기록한다.
- 작업 기간, 담당자, 브랜치, 커밋, 주요 산출물과 검증 결과를 함께 남긴다.

- 완료되지 않은 항목은 체크하지 않고 다음 작업으로 이월한다.
- 다른 역할의 작업 실적은 이 문서에 대신 작성하지 않고, R1 코드와 직접 맞닿는 인터페이스·협업 사항만 기록한다.

### 주요 구현 결과

#### 소스 구조와 역할 경계

- R1 전용 전투 로직은 `src/combat-core/` 아래의 `combat`, `enemies`, `player`, `waves`로 구성한다.
- 여러 역할이 함께 사용하는 `src/render/`, `src/scenes/`와 공용 진입점 `src/main.ts`는 최상위 위치를 유지한다.
- R2 계약 영역인 `src/spell/`도 최상위 위치를 유지한다.
- `ProtoScene`과 주문 렌더러는 `SpellSpec`을 참조하되 R2 스키마 구조를 변경하지 않는다.

#### 플레이어와 카메라

- WASD로 이동하며 대각선 입력 시 속도를 정규화한다.
- 카메라는 플레이어를 부드럽게 추적하고 HUD는 화면에 고정된다.
- 영창 입력 중에는 플레이어 이동을 차단하고 게임 진행 시간을 10%로 낮춘다.
- 플레이어 상태는 HP 100, 마나 100을 기준으로 관리하며 마나 비용과 글로벌 쿨다운을 검사한다.

#### 적과 피해 처리

- 추격자는 플레이어를 직선으로 추적하고 개체별 쿨다운으로 접촉 피해를 준다.
- 사수는 선호 거리를 유지하면서 3발 확산 탄막을 발사하고 접촉 피해도 준다.
- 대형 분열체는 처치 시 소형 분열체 2기로 나뉘며, 소형까지 모두 처치해야 웨이브가 끝난다.
- 기본 공격은 가장 가까운 적을 추적하고 적중 즉시 사라지며 관통하지 않는다.
- 주문 피해의 1차 공식은 `round(power)`이며, 폼별 최종 밸런스는 후속 작업으로 남겼다.

#### 웨이브와 주문 폼

- 웨이브 1: 추격자 3기
- 웨이브 2: 추격자 2기 + 사수 2기
- 웨이브 3: 추격자 2기 + 사수 2기 + 대형 분열체 1기
- 마지막 활성 적을 처치하면 웨이브 완료 또는 `ROOM CLEAR` 상태로 전환한다.
- `beam`은 목표 방향의 직선 범위를 관통하며 범위 안의 적을 각각 한 번 타격한다.
- `wave`는 넓은 파면이 전진하며 닿은 적을 각각 한 번 타격한다.
- `size`, `speed`, 주·보조 원소 팔레트를 두 폼의 크기·속도·색상에 반영한다.

### 임시 구현 및 후속 협의 항목

다음 내용은 GDD에 구체적인 값이나 동작이 없어 프로토타입 검증을 위해 임시로 정했다.

- 스크롤 카메라와 화면 2배 크기의 직사각형 월드
- 마나 회복량, 이동·공격 속도와 적별 피해 수치
- 기본 공격 사거리 `400px` 및 피해 `10`
- 1분 이내 3웨이브 검증을 위한 적 HP: 추격자 `30`, 사수 `20`, 대형 분열체 `40`, 소형 분열체 `10`
- 웨이브별 적 조합과 웨이브 사이 2초 대기
- `beam` 사거리·폭·관통 방식과 `wave` 사거리·폭·이동 속도
- 주문 상태이상과 넉백은 이번 범위에서 제외
- 주문명 공통 최대 길이 `30자`와 긴 이름 자동 줄바꿈

해당 값은 플레이테스트와 팀 합의 후 확정하거나 교체해야 한다.

- `EnemyControlState`가 적별 둔화 수명, 만료, 비중첩 갱신과 정리를 관리한다.
- 적 공통 `update` 계약에 이동 배율을 추가하되 공격·접촉 피해 쿨다운은 실제 전투 시간으로 유지한다.
- `control` 주문은 기존 폼의 적중 영역을 사용하지만 적에게 직접 피해를 주지 않는다.
- 둔화 중인 적에는 푸른색 상태 링을 표시하고, 만료·처치·방 전환 시 제거한다.

- 미지원 `status`도 이번 단계에서는 기본 둔화로 적용한다.
- `summon`은 플레이어 주변을 도는 마법 구체로 축소하고, 사거리 안의 가장 가까운 적에게 원소색 유도탄을 발사한다.
- 소환체는 활성 1개로 제한하고 방 종료·런 완료·플레이어 사망 시 정리한다.
- 소환 지속시간 `5~10초`, 피해 `power × 0.3`, 공격 간격 `1.2초`, 사거리 `400px`는 임시값이다.
- `cage`와 범용 `summon` 폼 렌더러는 후속 작업이다.
